Let's begin with a simple concept: One-way hashing functions.

A one-way hashing function is a simple mechanism that takes in something as an input and produces a unique hash value for it. The hashing function we will be using is called SHA-256.

Let's try it. Input any text into the box below and click the "Hash It!" button.

Now, imagine you have a simple zip archive with 4 files in it. The application is on the web somewhere. You need to download it and make sure that the file downloaded correctly. In other words, you have to verify that the files are not corrupted. How can you do this?

One way is using what is known as the checksum. The authors of the zip archive will use a known hashing function, such as SHA-256, to hash the contents of the archive and will make the resulting hash value public. This way after you download the zip archive, you can use the same hashing function on your own to hash the contents of the archive and compare your result with the one the authors got. If you get the same result, it means that the contents of the archive you downloaded must be the same as the contents of the one that the authors made available.
